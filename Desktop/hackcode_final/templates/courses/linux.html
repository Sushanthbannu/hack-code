Linux Basics

What is Linux?

Linux is an open-source, Unix-like operating system kernel that forms the foundation of many operating systems called Linux distributions or distros. Unlike proprietary operating systems, Linux is freely available and maintained by a global community of developers.

Linux is widely used in servers, desktops, mobile devices (Android), embedded systems, and supercomputers. Its stability, security, and flexibility have made it a preferred choice for developers and enterprises.

History of Linux

Linux was created by Linus Torvalds in 1991 while he was a student at the University of Helsinki. He developed the Linux kernel as a free alternative to the proprietary Unix systems.

The kernel was released under the GNU General Public License (GPL), allowing anyone to use, modify, and distribute the software. The combination of the Linux kernel with GNU tools created a fully functional operating system.

Over time, various distributions emerged, tailoring Linux for different purposes and users.

Linux Distributions

Linux distributions bundle the Linux kernel with various software, utilities, and package management systems. Popular distributions include:

  • Ubuntu: User-friendly and popular for desktops and servers.
  • Fedora: Cutting-edge technologies and Red Hat backing.
  • Debian: Stable and well-tested, often used as a base for other distros.
  • CentOS / Rocky Linux: Enterprise-grade, commonly used for servers.
  • Arch Linux: Rolling release with user customization focus.

Each distro serves different needs, from beginner-friendly to advanced customization.

The Linux Kernel

The Linux kernel is the core component of the operating system. It manages hardware, system resources, process scheduling, memory, and device drivers.

The kernel interacts directly with the hardware and provides services to user applications and software layers above it.

Linux kernel development is a collaborative effort by thousands of contributors worldwide, coordinated by Linus Torvalds.

Filesystem Structure

Linux follows the Filesystem Hierarchy Standard (FHS), organizing files and directories in a tree structure starting at the root /.

  • /bin - Essential binaries and commands
  • /etc - System configuration files
  • /home - User home directories
  • /lib - Shared libraries
  • /usr - User utilities and applications
  • /var - Variable data like logs and spool files
  • /tmp - Temporary files

Understanding the directory structure helps navigate and manage the system efficiently.

Basic Linux Commands

Here are some essential commands to get started:

  • ls - List directory contents
  • cd - Change directory
  • pwd - Print current directory path
  • cp - Copy files or directories
  • mv - Move or rename files
  • rm - Remove files or directories
  • mkdir - Create directories
  • cat - Display file contents
  • echo - Print text to the terminal
  • man - Show manual for commands

File Permissions

Linux uses a permission system to control access to files and directories. Permissions are divided into three categories:

  • Read (r): View contents
  • Write (w): Modify contents
  • Execute (x): Run as a program or script

Permissions are assigned to three types of users:

  • Owner: The user who owns the file
  • Group: A set of users assigned to a group
  • Others: Everyone else

Use the chmod command to change permissions and chown to change ownership.

chmod 755 script.sh
chown user:group file.txt

Process Management

Processes are running instances of programs. Linux offers tools to view and manage processes:

  • ps - List running processes
  • top - Interactive process viewer
  • kill - Send signals to processes, typically to terminate
  • htop - Enhanced interactive process viewer (needs installation)

Example to kill a process by PID:

kill 1234

Package Management

Package managers help install, update, and remove software. Different distros use different package managers:

  • APT (Debian/Ubuntu): sudo apt update && sudo apt install package-name
  • YUM/DNF (Fedora/CentOS): sudo dnf install package-name
  • PACMAN (Arch Linux): sudo pacman -S package-name

Shell Scripting Basics

Shell scripts automate tasks using a series of commands written in a file with .sh extension.

Example script:

#!/bin/bash
echo "Hello, Linux user!"
date

Make the script executable and run it:

chmod +x script.sh
./script.sh

Shell scripting can include variables, conditionals, loops, and functions.

Networking Basics

Linux provides powerful tools to manage and troubleshoot network connections:

  • ifconfig or ip addr - Show network interfaces
  • ping - Test connectivity to another host
  • netstat - Display network connections
  • ssh - Secure remote login

Security Tips

  • Regularly update your system with security patches
  • Use strong passwords and SSH keys for remote access
  • Configure a firewall (e.g., UFW) to limit open ports
  • Limit root user access and use sudo for administrative tasks
  • Monitor system logs for suspicious activity

Conclusion

Linux is a powerful and versatile operating system that forms the backbone of many technologies today. Whether you're a developer, system administrator, or just curious about open-source software, learning Linux opens up a world of possibilities.

Start exploring Linux today, and you'll find a supportive community and endless resources to help you on your journey.
